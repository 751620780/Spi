========================================================================
    控制台应用程序：Install 项目概述
========================================================================

应用程序向导已为您创建了此 Install 应用程序。

本文件概要介绍组成 Install 应用程序的每个文件的内容。


Install.vcxproj
    这是使用应用程序向导生成的 VC++ 项目的主项目文件，其中包含生成该文件的 Visual C++ 的版本信息，以及有关使用应用程序向导选择的平台、配置和项目功能的信息。

Install.vcxproj.filters
    这是使用“应用程序向导”生成的 VC++ 项目筛选器文件。它包含有关项目文件与筛选器之间的关联信息。在 IDE 中，通过这种关联，在特定节点下以分组形式显示具有相似扩展名的文件。例如，“.cpp”文件与“源文件”筛选器关联。

Install.cpp
    这是主应用程序源文件。

/////////////////////////////////////////////////////////////////////////////
其他标准文件:

StdAfx.h, StdAfx.cpp
    这些文件用于生成名为 Install.pch 的预编译头 (PCH) 文件和名为 StdAfx.obj 的预编译类型文件。

/////////////////////////////////////////////////////////////////////////////
其他注释:

应用程序向导使用“TODO:”注释来指示应添加或自定义的源代码部分。

/////////////////////////////////////////////////////////////////////////////
为了跨系统版本使用本程序，请将每一个项目的“配置属性->常规->平台工具集”选择成“Visual Studio 2015 - Windows XP (v140_xp)”（VS2015是这个，其他的vs版本请自己类比,这表示，最低支持XP）
	“配置属性->常规->MFC的使用”选择成“在静态库中使用 MFC”，则使得编译后的文件很大，但是在其它本本系统上不会因为缺少某些dll而无法运行。因为在其他dll中所需函数已经静态链接到文件中了
	“C/C++->代码生成->运行库”选择“多线程调试 (/MTd)”，选择这个编译后文件最大，因为这是静态(链接)多线程debug版本，若选择“多线程 (/MT)”编译出的文件较小一点，因为这是静态(链接)多线程release版本
	这样设置后会带来一些问题，比如编译报错，请使用预编译指令“#pragma comment (lib,"Advapi32.lib")”导入与代码相关的所有lib文件（这里以Advapi32.lib为例）
